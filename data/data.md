# Data Directory Overview

This `data` folder contains the core geospatial bundles provided by BhuMe, as well as the mathematically aligned predictions generated by our pipeline.

The dataset consists of two massive agricultural villages in Maharashtra:
- **`village_Malatavadi`** (~2,508 plots)
- **`village_Vadnerbhairav`** (~2,457 plots)

## File Structure per Village

Inside each village bundle, you will find the following core data files:

### Inputs (Raw Data)
- **`imagery.tif`**: The primary georeferenced satellite mosaic (RGB raster) showing the true physical layout of the fields, trees, and roads.
- **`boundaries.tif`**: A rough, algorithmically-generated probability map showing hints of visible boundaries. (Our algorithm uses this in tandem with Canny edges).
- **`input.geojson`**: The official, digitized plot geometry from government land records. These geometries are fundamentally misaligned due to historical paper-to-digital translation errors.
- **`example_truths.geojson`**: A small, hand-labeled subset of correctly aligned plots used strictly for objective evaluation and scoring.

### Outputs (Our Predictions)
- **`predictions.geojson`**: The final artifact produced by our memory-isolated multiprocessing pipeline. This file contains the exact same plots from `input.geojson`, but with their coordinate geometries mathematically shifted and rotated to align precisely with the physical edges seen in `imagery.tif`. It also contains our `confidence` scores and `status` flags (e.g. `Corrected` or `Flagged` for ambiguous boundaries).
- **`gpupred.geojson`**: An identical safety backup of the final predictions.
